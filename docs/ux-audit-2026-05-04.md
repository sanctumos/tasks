# Sanctum Tasks — UX audit (2026-05-04)

Method: Playwright walkthrough at **1440×900** (desktop) and **390×844**
(mobile), logged in as `admin`, fixtures of 3 projects + 5 tasks. Source
screenshots are in `tools/design-smoke/output/{desktop,mobile}/` (gitignored).

The plumbing is solid — auth, CSRF, audit log, API key, projects, tasks,
to-do lists, watchers. The **interaction model and information hierarchy**
are what feel clunky, not the engineering.

---

## What works

- **Login**, **Change password**, **API keys**, **MFA**: clean, focused,
  responsive. These pages don't need rework right now.
- **Workspace projects** (list): scannable table, "Open" affordance is clear.
- **Top nav**: collapsible, touch targets are now reasonable on mobile.
- **API key UX**: "Create new key" feels right; reveal-once + preview list
  matches what an operator expects.

## What's clunky (with concrete causes)

### 1. Tasks index is three pages stacked on one page
Order is **Filters → Create task → Task list**. By the time you scroll to
the task list (the *primary* job), you've passed two large forms.

- **Create task** is always expanded — for routine triage you don't need it.
- **Filter** is always expanded — for steady-state work you'd expect a
  compact bar (search + status + project).

### 2. Per-row inline edits add ~3 forms × 1 button each per task
Each row has separate Status, Priority, Assigned-to mini-forms with their
own **Save** button. Five tasks → fifteen Save buttons on one screen.
You also can't change two things at once.

- Selects don't auto-submit; nothing visually links the select to its Save.
- "Signals" column header (`C 0 / A 0 / W 0`) is jargon for
  Comments / Attachments / Watchers; readers shouldn't have to decode it.
- Project column is text, not a link to that project.
- "Created by", "Updated", and full "Due" timestamps eat horizontal space
  before the Actions column.

### 3. Tasks index on mobile renders the table inside `table-responsive`
The horizontal scrolling table on a 390-wide viewport produces clipped
selects (`to v`, `do v`, `low v`) and unreadable rows. Below tablet width
this should be a **card list**, not a wide table.

### 4. Task detail (`view.php`) duplicates and over-stacks
- Top card shows the title as **both** the card heading and the first row
  of the `<dl>` ("Title: …").
- Four large cards stacked top-to-bottom: **Read details → Update form →
  Danger Zone → Collaboration**. Empty Watchers/Attachments/Comments take
  a full card just to say "No watchers / No attachments / No comments."
- "Recurrence" / "Due at" are blank labels with no value.
- Update form repeats fields the read view already shows.

### 5. Users page mixes "user CRUD" with the **system audit log**
Two unrelated jobs are crammed onto one screen. The audit log is global,
not user-scoped, but it's parked under `/admin/users.php`. On mobile the
"Person kind" column truncates (`team_memb`).

### 6. Workspace project detail has empty containers
**Members**, **To-do lists**, and (implicitly) **Tasks in this project**
are split across three blocks; tasks-in-project aren't shown at all.
You currently have to leave this page and go filter `index.php` by name to
see the work that lives in the project.

### 7. Inconsistent navigation language
Three different "go back" labels live across pages:
- Tasks index → "Home" (links to root, not a meaningful home)
- Other admin pages → "Back to Tasks"
- Project detail → "All projects"

### 8. Root `/` is a half-public landing
A logged-in admin still lands on a basic "About this app" card with two
buttons. There's no reason a logged-in user shouldn't go straight to
`/admin/`.

### 9. Visual hierarchy: every section is a `card.shadow-sm`
Three to four equally-weighted cards per page means **nothing reads as
primary**. Whitespace + section headings would do most of this work for
free.

## Recommendations (proposed direction)

These are ordered by **highest UX leverage first**. None of them require
ripping out existing code; they reshape the information architecture and
let us delete chunks rather than add.

### A. Reframe the admin into three zones, not eight pages

- **Home (`/admin/`)**: a triage list, **not** a kitchen-sink page.
  - Compact filter bar across the top (Search · Status · Project · Mine).
  - **One** task list, no inline create form, no big filter card.
  - "+ New task" opens a modal (or a focused `/admin/new.php`).
- **Project workspace (`/admin/project.php?id=…`)**: this is the
  collaboration surface that's missing.
  - Header: project name, status, members avatars, "+ New task".
  - Tabs (or sections) under the header: **Tasks · Lists · Members ·
    Settings**.
  - The tasks tab shows that project's tasks **grouped by list/status**,
    not the global task list filtered after the fact.
- **Settings (`/admin/settings/…`)**: Users, API keys, MFA, password,
  audit log — each as its own page, separated from each other. The
  audit log moves out of `users.php` to `/admin/settings/audit.php`.

### B. Tasks list redesign

- Replace per-row mini-forms with **row click → side drawer** (or row →
  expanded inline edit) where you change status/priority/assignee/due
  in one form, **one Save**.
- Or, if we want to keep direct manipulation: **auto-save on change**
  for status/priority/assignee, with a small toast on success — kill
  the per-cell Save buttons.
- Add **status grouping** ("To do / Doing / Done") as the default view,
  with a flat-table toggle. This is the single biggest "feels modern"
  upgrade and is mostly CSS + a `GROUP BY` we already do.
- On **< md viewport**, render tasks as cards (title, status pill,
  priority chip, project, due, three-dot menu) instead of `table-responsive`.
- Replace the "Signals" column with three small **icons** (💬 / 📎 / 👁)
  with counts; hide when 0 to reduce noise.
- Make the **Project** cell a link to the project workspace.
- Rename column **Created by → Reporter**; drop ID column from default
  view (keep it in a hidden detail or compact `#1` chip).

### C. Task detail redesign

- One page, two-column desktop layout:
  - **Left (≈8/12):** title (editable in place), body, comments thread,
    activity feed.
  - **Right (≈4/12):** sticky metadata panel — Status, Priority, Due,
    Assignee, Project, Tags, Watchers, Attachments. Click any field to
    edit in place.
- Drop the giant duplicate **Update Task** card. The right rail *is* the
  editor.
- Show **Collaboration** sections only when they have content; empty
  states collapse to "Add a comment / attachment / watcher" CTAs.
- Move **Delete** into a small kebab menu in the header instead of a full
  red "Danger Zone" card.

### D. Project workspace (new screen, mostly composed of existing parts)

- **Tasks** within the project, grouped by **to-do list** (existing
  `todo_lists` model), with drag/drop or keyboard reorder later. This is
  where the Basecamp-shaped value lives without needing the Basecamp UI.
- Members section already exists — keep it but as a tab, not a card stack.
- Settings tab: name / description / client-visible / all-access / archive.

### E. Information architecture cleanup

- **Standardize "back" to a single pattern**: a back-arrow + canonical
  label (e.g. `← Projects`, `← Tasks`).
- **Standardize page titles**: Title Case for nav and h1. ("API keys",
  "MFA settings", "Change password" — pick one and apply.)
- **`/` for logged-in users → 302 to `/admin/`**, keep the marketing
  card only for anonymous visitors. Cheap, immediate win.
- **Audit log** out of users.php, into its own page, with filters
  (actor, action, entity).

### F. Visual polish (after IA settles)

- Replace 3-stacked-cards-per-page with **section headers + light
  dividers**, reserving cards for genuinely card-shaped data (a project
  card, a key card).
- Consistent **status pills** with color (todo gray, doing amber, done
  green, blocked red) and **priority chips** (low / normal / high /
  urgent). Reuse on list, detail, and project board.
- Tighten table cell padding on dense list views; grow it on detail.

## What I'd ship in what order

1. **Quick wins (1 short pass):**
   - Auto-save inline status/priority/assignee — kill the Save buttons.
   - "Signals" → icons; Project cell → link.
   - Logged-in `/` → redirect to `/admin/`.
   - Move audit log off `users.php`.
   - Standardize "back" labels and page-title casing.
2. **Tasks list v2 (medium pass):**
   - Compact filter bar + collapsible "+ New task" modal.
   - Mobile card layout for `< md`.
   - Group-by-status default view.
3. **Task detail v2 (medium pass):**
   - Two-column layout, sticky metadata rail, in-place edit, collapsing
     empty collaboration blocks.
4. **Project workspace v2 (the new home for collaboration):**
   - `/admin/project.php?id=…` with Tasks (grouped by list) · Lists ·
     Members · Settings tabs. This is where to-do lists become useful and
     where multi-person work lives.

Items 1–2 alone should make the admin feel **noticeably less clunky** for
day-to-day use. Item 4 is what gives Sanctum Tasks its own product shape
(rather than "a flat task table with rich admin").
