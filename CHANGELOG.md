# Changelog

## [Unreleased]

### Changed

- **Tasks require a todo list (`list_id`).** Creating a task now requires an explicit **`list_id`** (GET `/api/list-todo-lists.php?project_id=…`). You may send **`list_id` alone** (project is inferred from the list) or **`project_id` + `list_id`** together (must match). DB bootstrap seeds a **`General`** list for every `projects` row that had none, then backfills **`tasks.list_id`** where **`project_id`** was set — same SQL as `tools/migrations/sanctum_004_task_list_required.sql`. **`POST /api/update-task.php`** rejects clearing **`list_id`**; changing **`project_id`** without **`list_id`** assigns the first list in the target project (must exist). Admin: task rail **List** dropdown (`/admin/view.php`), **New task** modals on `/admin/` and `/admin/project.php` require a list; project pages auto-create **General** when the viewer can manage and no lists exist.

### Added

- **Documents** — long-form markdown reference material with its own discussion thread, attached to a directory project. Modeled on Basecamp's *Docs* pattern: write a spec, runbook, decision record, or onboarding note as markdown, and let teammates reply below it. Schema: `documents` (`project_id` FK, `title`, `body`, `status`, `created_by_user_id`, timestamps) + `document_comments` (per-doc thread). Idempotent migration in `initializeDatabase()` and `tools/migrations/sanctum_003_documents.sql`. New PHP DB layer in `public/includes/functions.php`: `createDocument`, `getDocumentById`, `listDocumentsForUser` (with optional `project_id` filter), `updateDocument`, `deleteDocument`, `listDocumentComments`, `addDocumentComment`, plus `userCanAccessDocument` / `userCanManageDocument` access checks (project access + role-based management). API endpoints: `GET /api/list-documents.php`, `GET /api/get-document.php`, `POST /api/create-document.php`, `POST /api/update-document.php`, `POST /api/delete-document.php`, `GET /api/list-document-comments.php`, `POST /api/create-document-comment.php`. Admin UI: top-level **Docs** nav item; `/admin/docs.php` (cross-project list with per-project filter), `/admin/doc.php?id=N` (markdown render + Discussion + composer + inline title/body edit + permalink for cross-linking from task discussions), `/admin/doc-create.php`, plus session-auth POST endpoints `/admin/doc-update.php`, `/admin/doc-comment.php`, `/admin/doc-delete.php`. **Docs** tab added to the project workspace (`/admin/project.php?tab=docs`). Coverage: 5 PHPUnit unit tests (`tests/php/Unit/DbDocumentLifecycleTest.php`) + 3 pytest integration tests (`tests/integration/test_php_documents_integration.py`); design verification harness at `tools/design-smoke/docs_verify.py` covers the docs list and a single-doc view at desktop + mobile.

### Changed

- **Task detail page — markdown, inline timestamps, recurrence builder.** Description body and comment bodies now render **Markdown** via Parsedown 1.7.4 (vendored at `public/includes/lib/Parsedown.php`, MIT) in safe mode with HTML escaping and auto-linked URLs — bold, italic, headings, lists, fenced code, blockquotes, tables, all rendered in-place; helper `st_markdown()` in `public/admin/_helpers.php`. Discussion items now display the **absolute UTC timestamp inline** alongside the relative time (e.g. `May 6 04:18 UTC · 2h ago`); same treatment for the page header `opened` / `updated` chips. The free-text **Recurrence** field is replaced by a structured **builder modal**: pick frequency (Daily/Weekly/Monthly/Yearly/Custom), interval, weekdays for Weekly, day-of-month for Monthly, and end condition (Never / After N occurrences / On date) — with a live human summary and the underlying `RRULE` shown for verification. Existing rules are parsed back into form state on open (JS in `public/assets/admin.js`: `parseRRule` / `buildRRule` / `humanizeRRule`); PHP-side `st_humanize_rrule()` renders the same summary on the rail button. CSS bumped to `?v=5`, JS to `?v=4`.
- **Task detail page redesign** (`/admin/view.php`) — the cramped *Activity* card is gone. Comments now render as a real **Discussion** thread (avatar + author + tooltip-bearing relative timestamp + multiline body, URLs auto-linked), with a **comment composer right on the page** posting through new admin endpoint `/admin/comment.php` (CSRF, session-auth wrapper around `addTaskComment`). **Watch / Unwatch** is a first-class header button (new `/admin/watch.php`). Header is rebuilt as labeled chips (id, project link or *orphan task* warning, opened/updated with absolute-time tooltips) plus inline title rename. Description now has inline edit (no scrolling past the discussion to fix typos). Metadata rail’s *Project* control became a **dropdown bound to `project_id`** so editing it can no longer re-orphan a task; *Tags* row is now editable autosave (one source of truth); *Recurrence* moves to the rail. Attachments split into their own card. New `_helpers.php` utilities: `st_initials`, `st_avatar_html`, `st_format_comment_body`, `st_absolute_time_attr`. CSS bumped to `?v=4`, JS to `?v=3`. Visual verification harness added (`tools/design-smoke/task_view_verify.py`).
- **Repository split:** the FastAPI stack (`api_python/`) and its pytest suite moved to **`sanctumos/py-tasks`**. This repo keeps PHP (`public/`), Composer/PHPUnit, `tasks_sdk/`, and `smcp_plugin/`. CI no longer installs FastAPI deps here.

### Fixed

- **PHPUnit-style integration drift** — `test_php_api_admin_integration.py` now matches the current public landing (`API health`), settings-based password change (`settings.php?tab=password`), **Users** heading, and MFA under **settings** (`mfa_generate` / `mfa_enable` / `mfa_disable`) instead of legacy `/admin/mfa.php` POSTs.

- **Forced password change / multihost login** — `requireAuth()` now treats password-change allow paths by **suffix** (`…/admin/settings.php`, etc.) so `SCRIPT_NAME` with a subdirectory prefix no longer traps users in redirect loops or `.Err` pages. Redirect targets use **relative** `settings.php?tab=password`; legacy `change-password.php` and successful login with `must_change_password` go straight to the Password tab. Login “already signed in” / default landing use relative `index.php` for subpath installs.

### Added

- **Organizational controls (admin UI + enforcement)** — `users.limited_project_access` (idempotent column; see `tools/migrations/sanctum_002_limited_project_access.sql`). **Admins** see the full org project directory; **managers** do by default but can be toggled to **limit projects** (same rules as members: `project_members` + `projects.all_access`). **Organizations** admin (`/admin/organizations.php`): create/rename orgs, counts. **Users** (`/admin/users.php`): assign organization, limit toggle, link to **project membership** editor (`/admin/user-projects.php`). Task list/search/API scope (`listTasks` 4th argument / merged API user row) and **task read access** align with workspace visibility (`userCanAccessTaskForViewer`). Navbar adds **Organizations**. FastAPI mirror updated (`user_has_unrestricted_org_directory_access`, `list_tasks` scope).
- **Project CRUD, membership, pins, to-do lists, task linkage** — `GET/POST` directory project update, project members, `todo_lists` + `tasks.list_id`, `user_project_pins`, client-only directory filter (`client_visible` + `person_kind=client`), `backfillTaskProjectIdsFromLegacyNames()` in PHP. Tasks: `project_id` / `list_id` on create, update, and list filters; `directory_project` on task JSON when joined. New admin page `workspace-project.php` (from `workspace-projects.php`). Idempotent DDL in `config.php` / `api_python/db.py` and migration notes updated.
- **Organizations & project directory** — SQLite `organizations`, `projects`, `project_members`; `users.org_id` and `users.person_kind` (`team_member` \| `client`); `tasks.project_id` (nullable FK) for later backfill from legacy `tasks.project` string. Default org **Default** + user org linking on bootstrap. API: `GET /api/list-organizations.php`, `GET /api/list-directory-projects.php`, `POST /api/create-directory-project.php`. PHP + FastAPI mirror; SDK `list_organizations`, `list_directory_projects`, `create_directory_project`; `create_user` accepts optional `org_id` / `person_kind`. Admin: **Users** shows org + person kind; **Workspace projects** (`/admin/workspace-projects.php`) lists and creates directory projects. `createUser` self-heals default org before insert. Idempotent migration notes in `tools/migrations/sanctum_001_organizations_and_projects.sql`. **Deploy:** server must `git pull` (or your usual sync) to this `main` — the browser UI will not change until the new PHP is on the host.
- **Heartbeat setup wizard** — `scripts/setup_heartbeat.sh`: interactive bash wizard to configure an open-claw heartbeat worker. Asks for agent name, sanctum folder, Tasks creds (type now or path to existing .env), heartbeat project, optional worker user id, interval, and whether to add a crontab entry. Writes `run_heartbeat.sh` (bash + curl) and `.env.heartbeat` under `sanctum/agents/<name>/`. See [docs/HEARTBEAT.md](docs/HEARTBEAT.md) and [docs/HEARTBEAT_WIZARD_CONTEXT.md](docs/HEARTBEAT_WIZARD_CONTEXT.md).
